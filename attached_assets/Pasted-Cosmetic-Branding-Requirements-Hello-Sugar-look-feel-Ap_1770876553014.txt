Cosmetic / Branding Requirements (Hello Sugar look & feel)

Apply a consistent visual design so this tool feels like an internal Hello Sugar product (hellosugar.salon vibe): modern, clean, feminine, bright, confident, high-trust. This is purely UI/UX styling and should not change functionality.

Brand personality
- Friendly, upbeat, polished
- Not “corporate enterprise gray”
- Lots of whitespace, rounded shapes, soft shadows, smooth interactions

Color palette (use as CSS variables)
- Primary (Hello Sugar Pink): #FF4FA3
- Primary hover: #E83E8C
- Primary pressed: #CC2F76
- Primary subtle background: #FFF0F7

- Secondary (Warm Coral/Peach): #FF8A5C
- Secondary hover: #F77747
- Secondary subtle background: #FFF2EC

- Accent (Sunshine): #FFC857
- Accent subtle background: #FFF7E0

- Neutrals:
  - Ink / Text: #1F2937
  - Muted text: #6B7280
  - Border: #E5E7EB
  - Background: #FAFAFC
  - Card surface: #FFFFFF
  - Elevated shadow: rgba(17, 24, 39, 0.08)

- Status colors:
  - Success: #16A34A (bg: #EAF7EE)
  - Warning: #F59E0B (bg: #FFF4DE)
  - Error: #DC2626 (bg: #FDECEC)
  - Info: #2563EB (bg: #EAF2FF)

Typography
- Use a clean modern sans font stack that resembles the Hello Sugar site:
  - Preferred: "Inter" (Google Font) or "Montserrat" as a fallback option
  - Fallback stack: system-ui, -apple-system, Segoe UI, Roboto, Arial, sans-serif
- Type scale:
  - Page title: 28–32px, weight 700
  - Section headers: 18–20px, weight 600
  - Body: 14–16px, weight 400–500
  - Small labels: 12–13px, weight 500, letter spacing slightly increased
- Use sentence case labels, not ALL CAPS.

Layout + spacing
- Mobile-first, responsive.
- Max content width: 1100–1200px for admin pages; esthetician flow can be narrower (600–760px).
- Use cards for grouping content:
  - white background, 14–18px border radius
  - subtle border (#E5E7EB)
  - soft shadow (light, not heavy)
- Generous whitespace: 16–24px padding inside cards, 16px vertical rhythm.

Buttons (make these consistent everywhere)
- Primary button:
  - background: Primary pink (#FF4FA3)
  - text: white
  - radius: 12px
  - height: 44px minimum
  - padding: 12px 18px
  - font-weight: 600
  - hover: darken to #E83E8C
  - focus ring: 2–3px ring in pink with subtle opacity
- Secondary button:
  - background: white
  - border: 1px solid #E5E7EB
  - text: #1F2937
  - hover: light pink tint background (#FFF0F7)
- Destructive button:
  - background: #DC2626
  - text: white
  - hover: #B91C1C

Inputs + dropdowns
- Rounded corners (10–12px)
- Height 44px
- Border #E5E7EB
- On focus:
  - border becomes primary pink
  - soft focus ring
- Placeholder text in muted gray
- Errors show inline under input with red text and subtle red border

Tables (admin portal)
- Clean, airy tables with:
  - sticky header row
  - zebra striping very subtle (#FAFAFC alternating)
  - rounded container with border
- Filters should be “pill” style chips or a clean filter bar.

Navigation
- Top bar with app name “CashControl” + a small sublabel “Hello Sugar”
- Use pink as highlight for active nav
- Left sidebar optional; if used, keep it minimal with icons + labels.

Badges + status chips
- Rounded pill chips for statuses:
  - Variance: error style
  - Missing submission: warning style
  - OK / reconciled: success style
  - Receipt submitted: info style

Micro-interactions
- Subtle hover states on clickable cards/rows
- Toast notifications styled to match palette
- Smooth transitions (150–250ms)

Brand details
- Include a small “Hello Sugar” wordmark text in the footer or login screen (no need to fetch an actual logo unless provided).
- Use friendly empty states:
  - “All good — no variances right now.”
  - “Waiting on end-of-shift submissions.”

Do not over-style
- Keep it clean and modern. Avoid gradients everywhere. Use color as accents and for primary actions.

Deliverable expectation
- Apply this design system consistently across:
  - Esthetician flow screens
  - Admin portal dashboards
  - Alerts log
  - Receipt upload UI
  - Cash collection workflow

Implement as a centralized theme (CSS variables / design tokens) so it’s easy to update later.
